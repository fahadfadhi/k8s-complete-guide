# Kubernetes Dashboard — Setup, Access, and Exploration Guide

## 1. Introduction

The **Kubernetes Dashboard** is a web-based UI that lets you:
- View cluster resources visually
- Manage Deployments, Pods, Services, ConfigMaps, Secrets
- Monitor cluster health
- Access logs, exec, events
- View nodes, workloads, namespaces
- Perform CRUD operations safely

This guide includes:
✔ Installation  
✔ User creation (with secure access)  
✔ Token retrieval  
✔ Access through kubectl proxy  
✔ Dashboard navigation overview  
✔ RBAC best practices  

---

# 2. Install Kubernetes Dashboard (Official Recommended Version)

Apply the official manifest:

```
kubectl apply -f https://raw.githubusercontent.com/kubernetes/dashboard/v2.7.0/aio/deploy/recommended.yaml
```

Verify installation:

```
kubectl get pods -n kubernetes-dashboard
kubectl get svc -n kubernetes-dashboard
```

---

# 3. Create Admin User (ServiceAccount + ClusterRoleBinding)

### Create file: `admin-user.yaml`

```yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: admin-user
  namespace: kubernetes-dashboard
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: admin-user-binding
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
- kind: ServiceAccount
  name: admin-user
  namespace: kubernetes-dashboard
```

Apply:

```
kubectl apply -f admin-user.yaml
```

---

# 4. Get Login Token

```
kubectl -n kubernetes-dashboard create token admin-user
```

Copy the token → you will paste it into the dashboard login page.

---

# 5. Access the Dashboard

Start proxy:

```
kubectl proxy
```

Open:

```
http://localhost:8001/api/v1/namespaces/kubernetes-dashboard/services/https:kubernetes-dashboard:/proxy/
```

# 6. Dashboard Overview (Key Panels)

### **1. Cluster → Nodes**
- Node status & conditions  
- CPU/memory usage  
- Pods running per node  

### **2. Workloads**
Includes:
- Deployments  
- ReplicaSets  
- Pods  
- StatefulSets  
- DaemonSets  
- Jobs & CronJobs  

For each workload you can:
- View logs  
- Edit YAML  
- Exec into Pod  
- Restart Pods (by updating deployment)  

### **3. Discovery & Load Balancing**
Shows:
- Services  
- Endpoints  
- Ingresses  

### **4. Config & Storage**
- ConfigMaps  
- Secrets  
- StorageClasses  
- PersistentVolumeClaims  
- PersistentVolumes  

### **5. RBAC**
- ServiceAccounts  
- Roles  
- ClusterRoles  
- Bindings  

---

# 7. Secure Access Recommendations

✔ **DO NOT expose Dashboard publicly**  
✔ Always use **RBAC-controlled tokens**, NEVER static admin tokens  
✔ Use **OIDC** for production login (Dex, AWS IAM, Azure AD, Google Auth)  
✔ Use **read-only roles** for developers  
✔ Disable exec/logs if unnecessary  

---

# 8. Create Read‑Only Access (Safer for Dev)

```yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: readonly
  namespace: kubernetes-dashboard
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: readonly-binding
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: view
subjects:
- kind: ServiceAccount
  name: readonly
  namespace: kubernetes-dashboard
```

Generate token:

```
kubectl -n kubernetes-dashboard create token readonly
```

# 9. Uninstall Dashboard

```
kubectl delete -f https://raw.githubusercontent.com/kubernetes/dashboard/v2.7.0/aio/deploy/recommended.yaml
```


```
kubectl logs -n kubernetes-dashboard -l k8s-app=kubernetes-dashboard
```

### List service:
```
kubectl get svc -n kubernetes-dashboard
```
